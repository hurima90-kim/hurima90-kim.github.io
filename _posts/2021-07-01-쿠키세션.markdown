<h3>📌 왜, Cookie와 Session을 사용하나요?</h3>
서버와 클라이언트가 통신을 할 때 통신이 연속적으로 이어지지 않고 한 번 통신이 되면 끊어집니다.<br>
따라서 서버는 클라이언트가 누구인지 계속 인증을 해주어야 하는데 매번 인증을 한다는 것은 상당히 귀찮고 번거롭습니다.<br>
또한 웹페이지의 로딩속도를 느리게 만드는 요인중에 하나이기도 합니다. <br>
이러한 번거로움(클라이언트와의 정보 유지를 위한 인증)을 해결하기 위해 쿠키와 세션을 사용하는 것 입니다.<br><br>

![쿠키](https://user-images.githubusercontent.com/66353903/146675924-ac2be4a4-ab9f-4b32-b453-d719e97446c9.jpeg)

<h3>📌 Cookie</h3>
HTTP의 일종으로 사용자가 어떠한 웹 사이트를 방문할 경우, 그 사이트가 사용하고 있는 서버에서 사용자의 컴퓨터에 저장하는 작은 기록 정보 파일입니다.<br>
HTTP에서 클라이언트의 상태 정보를 클라이언트PC에 저장하였다가 필요시 정보를 참조하거나 재사용 할 수 있습니다.<br><br>

<h4>🔸 Cookie의 특징</h4>
1. 이름, 값, 만료일(저장 기간 설정), 경로 정보로 구성되어 있다.<br>
2. 클라이언트에 총 300개의 쿠키를 저장할 수 있다.<br>
3. 하나의 도메인 당 20개의 쿠키를 가질 수 있다.<br>
4. 하나의 쿠키는 4KB(=4096Byte)까지 저장이 가능하다.<br><br>

<h4>🔸 Cookie의 동작 순서</h4>
1. 클라이언트가 페이지를 요청 (사용자가 웹 사이트 접근)<br>
2. 웹 서버는 쿠키를 생성한다.<br>
3. 생성한 쿠키에 정보를 담아 HTTP 화면을 돌려줄 때, 클라이언트에게 같이 돌려준다.<br>
4. 넘겨 받은 쿠키는 클라이언트 PC에 저장하고 있다가 다시 서버에 요청할 때 요청과 쿠키를 함께 전송한다.<br>
5. 재방문시 클라이언트 PC에 해당 쿠키가 있다면, 요청 페이지와 쿠키를 함께 전송한다.<br><br>

<h4>🔸 Cookie의 사용예시</h4>
1. 방문했던 사이트에 재방문시 아이디와 비밀번호 저장, 자동입력<br>
2. 팝업창이 나타났을 경우 '하루동안 이 창을 열지 않음' 또는 '일주일간 다시 보지 않기'<br>
3. 쇼핑몰의 장바구니 기능<br><br>

<h3>📌 Session</h3>
일정 시간동안 같은 사용자(브라우저)로부터 들어오는 일련의 요구를 하나의 상태로 보고, 그 상태를 일정하게 유지시키는 기술이다.<br>
여기서 일정 시간은 방문자가 웹 브라우저를 통해 웹 서버에 접속한 시점으로부터 웹 브라우저를 종료하여 연결을 끝내는 시점을 말한다.<br>
즉, 방문자가 웹 서버에 접속해 있는 상태를 하나의 단위로 보고 그것을 세션이라고 한다.<br><br>

<h4>🔸 Session의 특징</h4>
1. 웹 서버에 웹 컨테이너의 상태를 유지하기 위한 정보를 저장한다<br>
2. 웹 서버의 저장되는 쿠키(= 세션 쿠키)<br>
3. 브라우저를 닫거나, 서버에서 세션을 삭제했을때만 삭제가 되므로, 쿠키보다 비교적 보안이 좋다.<br>
4. 저장 데이터에 제한이 없다.(서버 용량이 허용하는 한...)<br>
5. 각 클라이언트 고유 Session ID를 부여한다. Session ID로 클라이언트를 구분하여 각 클라이언트 요구에 맞는 서비스 제공한다.<br><br>

<h4>🔸 Session의 동작 순서</h4>
1. 클라이언트가 서버로 접속(요청)을 시도한다.<br>
2. 서버는 접근한 클라이언트의 Request-Header field인 Cookie를 확인하여 클라이언트가 해당 session-id를 보냈는지 확인한다.<br>
3. 만약 session-id가 존재하지 않는다면 서버는 session-id를 생성해 클라이언트에게 돌려준다.<br>
4. 서버에서 클라이언트로 돌려준 session-id를 쿠키를 사용해 서버에 저장한다(쿠키 이름은 JSESSIONID).<br>
5. 클라이언트는 재접속 시, 이 쿠키(JSESSIONID)를 이용하여 session-id 값을 서버에 전달한다.<br><br>

<h4>🔸 Session의 사용예시</h4>
1. 화면이 이동해도 로그인이 풀리지 않고 로그아웃하기 전까지 유지<br><br><br>

<h3>📝 마무리</h3>
쿠키로 노출시켜서는 안될 정보들이 있고, 세션을 남발하면 서버에 부담이 되어 과부하가 일어나기 때문에<br>
웹을 설계할 때는 이 정보는 쿠키에 저장할 지 세션에 저장할 지 적절한 판단을 내릴 수 있어야 합니다.
